# RequestKit

RequestKit 是一个 Manifest V3 Chrome 扩展，用来按站点或为所有站点的后续网络请求设置自定义请求头。

## 功能

- 自动识别当前标签页站点
- 支持「按站点」和「所有站点」两种互斥模式
- 按站点模式会为每个主机名保存独立规则，并应用到该站点标签页内的全部后续请求
- 所有站点模式会把一套共享请求头应用到全部 HTTP / HTTPS 站点
- 在紧凑的「所有站点规则」浮窗中搜索、筛选和集中管理全部站点
- 总开关与单条规则开关
- 新增、编辑、删除请求头
- 使用 `declarativeNetRequest` 即时同步标签页会话规则
- 规则保存在 `chrome.storage.local`，刷新页面后仍然存在

## 本地开发

```bash
npm install
npm run dev
```

浏览器中的开发预览会显示一组演示规则，方便直接检查界面和交互。正式扩展环境不会自动添加这些演示规则。

## 构建与安装

```bash
npm test
npm run build
```

然后打开 `chrome://extensions`：

1. 开启「开发者模式」。
2. 点击「加载已解压的扩展程序」。
3. 选择项目下的 `dist` 目录。

## 说明

RequestKit 只影响规则启用后发出的请求，已经完成的请求不会被改变。当前版本需要站点访问权限来应用请求头规则；按站点模式通过当前标签页的主机名选择配置，并把这组请求头应用到该标签页内的页面、第三方 CDN、脚本、样式、图片和接口等后续请求。所有站点模式则会把共享规则装载到每个 HTTP / HTTPS 标签页中。切换模式不会删除另一种模式下保存的配置。
